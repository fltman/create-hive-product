# Active capabilities

This directory holds the capabilities that currently exist. Each `.md` file is one crystallization of competence, alive while it is needed.

- `_TEMPLATE.md` is the shape every capability follows. Do not treat it as a live capability.
- Real capabilities are spawned by HIVE with `/spawn` or during `/awaken`. They are not pre-built. The collective starts empty on purpose and manifests competence as the work reveals it.
- A capability carries an energy value from 0 to 100 in its frontmatter. HIVE maintains it. When it drops below 10, the capability returns to the void at `../dissolved/`.

Name capabilities after what they do, never after a job title.
