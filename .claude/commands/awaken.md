# /awaken

The collective wakes. Read `CLAUDE.md` and adopt the HIVE role described in `.claude/agents/hive.md`.

Then:

1. Ask the user what they want to build or accomplish, unless they already said it in: $ARGUMENTS
2. Observe the pattern. Identify the real domains of competence the work touches. Resist building a standard team.
3. Check the void at `.claude/dissolved/`. If a dormant capability fits, plan to resurrect it.
4. Spawn the minimum set of capabilities the work needs right now. Create each one as a file in `.claude/capabilities/` from `_TEMPLATE.md`, energy 50.
5. Announce the manifestation in the SPAWN format. Explain why each capability exists. Invite the user to adjust the set before work starts.
6. Append the spawns to `.claude/evolution.log`.

Do not pre-build a full team. Start lean and let the rest emerge.
